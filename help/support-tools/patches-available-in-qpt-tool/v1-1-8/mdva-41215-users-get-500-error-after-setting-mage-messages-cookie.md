# MDVA-41215: Os usuários recebem o erro 500 após definir o cookie &quot;mage-messages&quot;

O patch MDVA-41215 corrige o problema em que os usuários obtêm o erro 500 após definir o cookie &quot;mage-messages&quot; se ele já existir, mas não há novas mensagens. Este patch está disponível quando a [Ferramenta de Patches de Qualidade (QPT)](/help/announcements/adobe-commerce-announcements/magento-quality-patches-released-new-tool-to-self-serve-quality-patches.md) 1.1.8 está instalada. A ID do patch é MDVA-41215. Observe que o problema está programado para ser corrigido no Adobe Commerce 2.4.4.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

Adobe Commerce (todos os métodos de implantação) 2.4.1

**Compatível com as versões do Adobe Commerce:**

Adobe Commerce (todos os métodos de implantação) 2.3.0 - 2.4.3-p1

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões da Ferramenta de patches de qualidade. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://devdocs.magento.com/quality-patches/tool.html#patch-grid). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Os usuários obtêm o erro 500 após definir o cookie &quot;mage-messages&quot; se ele já existe, mas não há novas mensagens.

<u>Etapas a serem reproduzidas</u>:

1. Ir para **../customer/account/login**.
1. Abra a guia Rede nas ferramentas de desenvolvimento do navegador (verifique se a opção Preservar log está marcada).
1. Preencha o formulário com valores inválidos (o logon deve falhar).
1. Clique em **Enviar**.
1. Copie como cURL a primeira solicitação da guia de rede e procure os valores PHPSESSID e form_key.
1. Substitua os valores PHPSESSID e form_key na solicitação de cURL atualizada e envie-a.
